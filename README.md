# 🏥 中医古籍智能辅助诊断系统 (TCM RAG Agent)

这是一个专为中医师和传统医学研究者设计的本地化智能辅助诊疗与文献查证系统。
该系统允许用户使用**现代自然语言（白话文）**描述患者症状，系统底层将自动映射至中医古语，并基于完全私有化部署的**混合 RAG（检索增强生成）**架构，在一秒内于《黄帝内经》、《伤寒杂病论》等海量古籍中进行精准溯源。所有诊断与方剂推荐均强绑定古籍原文出处，彻底杜绝医疗大模型常见的“幻觉”。

---

## 🛠 系统要求与前置准备

本系统 V0 阶段基于纯本地运行架构，绝不向外部网络传输任何敏感数据。
- **操作系统:** macOS / Linux / Windows
- **Python:** Python 3.10 或以上版本
- **硬件推荐:** 推荐拥有至少 8GB 统一内存的 Mac (M系列芯片) 或配备 8GB 以上显存的 NVIDIA 显卡 PC（用于流畅运行本地大模型）。
- **核心底层组件:** [Ollama](https://ollama.com/) (必须预先安装)

---

## 🚀 快速安装指南

### 1. 安装本地大模型基座 (Ollama)
系统的大模型推理（翻译、问答）与向量化（Embedding）强依赖 Ollama。
1. 前往 [Ollama 官网](https://ollama.com/) 下载并安装对应系统的客户端。
2. 打开您的终端 (Terminal)，执行以下命令下载我们系统依赖的两个核心模型：
```bash
# 下载主力对话与翻译大模型（默认使用 qwen3.5，也可根据硬件替换为 deepseek-r1）
ollama pull qwen3.5:latest

# 下载轻量级向量化模型
ollama pull nomic-embed-text:latest
```

### 2. 配置 Python 运行环境
将本项目代码下载到本地后，在终端进入代码目录，并创建虚拟环境：

```bash
# 1. 进入项目根目录
cd /path/to/your/project/Coding

# 2. 创建并激活虚拟环境 (推荐)
python3 -m venv venv
source venv/bin/activate  # Windows 用户使用: venv\Scripts\activate

# 3. 安装依赖包
pip install -r requirements.txt
```

---

## 📚 数据入库与初始化

在系统能回答问题之前，您需要建立“私人中医知识库”。

1. **放入古籍文件**：将您收集的古籍文本（支持 `.txt`, `.pdf`）放入项目的 `data/` 目录中。*(系统已内置部分测试医案)*
2. **执行摄取管道**：该脚本将自动切分文档、调用大模型进行古文到白话文的批量翻译，并生成双路向量索引存入本地 ChromaDB 数据库。
```bash
# 执行全量知识库构建
python run_ingestion.py
```
*(注：如果古籍文本极大，翻译阶段可能需要数十分钟。如果只想快速体验检索原文，可追加 `--skip-translate` 参数跳过白话文注释的生成。)*

---

## 🩺 运行诊断系统

知识库构建完毕后，即可启动可视化的医生工作站前端：

```bash
# 确保在虚拟环境激活状态下执行
streamlit run src/frontend/app.py
```

执行后，浏览器会自动弹出工作台网页（通常为 `http://localhost:8501`）。
您可以在左侧直接输入如：“患者发烧、怕冷、头痛、无汗，该怎么辨证？”，右侧的**古籍溯源面板**会立刻为您提供精确的出处证明。

## 📁 目录结构说明

```text
├── data/                    # 原始中医古籍存放目录 (.txt, .pdf)
├── ontology/                # 知识图谱与古今术语桥接映射文件
├── chroma_db/               # 自动生成的本地向量数据库
├── src/
│   ├── api.py               # FastAPI 后端服务入口
│   ├── config.py            # 全局配置文件 (可在此修改大模型名称或向量搜索阈值)
│   ├── ingestion/           # 知识摄取层 (分块、翻译、向量化)
│   ├── retrieval/           # 意图理解与混合检索层
│   ├── generation/          # 链式思考 (CoT) 与诊断生成层
│   └── frontend/
│       └── app.py           # Streamlit 双分屏医生 UI 前端
├── requirements.txt         # 依赖清单
└── run_ingestion.py         # 知识库构建入口脚本
```
